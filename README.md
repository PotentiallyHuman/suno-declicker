# suno-declicker

Removes the click/tick audio artifacts found in songs generated by **Suno AI v5.5** (and earlier versions).

## The problem

Suno v5.5 occasionally injects ultra-short broadband spikes into generated audio — audible as a sharp click or tick, typically on vocal consonants. They are under 0.3ms long, very loud relative to surrounding audio, and broadband (all frequencies spike at once).

## How it works

**Detection** uses three filters:
1. **Local amplitude ratio** — finds samples significantly louder than surrounding music (quiet passages with clicks are still caught)
2. **Spectral fingerprint** — compares the spike's frequency profile against 3 confirmed Suno v5.5 artifacts using cosine similarity. Musical transients (hi-hats, snares, guitar picks) score low and are skipped.
3. **Duration gate** — Suno artifact clicks are under 15 samples (~0.3ms); real musical transients are longer

**Repair** uses a band-split approach:
- **Below 2 kHz** — cubic spline interpolation from the original's own surrounding samples (preserves the low-end perfectly, no phase issues)
- **Above 2 kHz** — the vocal + instrumental stems replace the broadband click energy

This means the original audio is used everywhere except the handful of samples that contain the click. The original file is **never modified**.

## What you need

For best results you need three files — all downloadable from Suno:

| File | What it is |
|------|------------|
| **Original** | The full song mix (mp3 or wav) |
| **Vocal stem** | Vocals with effects — download from Suno's stem export. Use the version *with* effects so the tone matches the mix at the click location. |
| **Instrumental stem** | Instruments with effects — same source |

Place all three files in the same folder before running.

> **Why stems?** The click artifacts appear in the final mix but not always in the individual stems. At each detected click location the script blends in the clean stem audio for a fraction of a millisecond, then returns to the original. Everything else is untouched.
>
> If you don't have stems, the script still works — it falls back to cubic spline repair for all frequencies.

## Usage

### Interactive (recommended)

Just run the script with no arguments. It lists all audio files in the current folder and walks you through selecting each file:

```
python declicker.py
```

```
  ┌─────────────────────────────────────────┐
  │          suno-declicker setup           │
  └─────────────────────────────────────────┘

  Place your files in the same folder and run this script.
  The original is NEVER modified — a clean copy is saved.

  Select the ORIGINAL full mix:
      1.  My Song (Remaster).mp3
      2.  My Song-instrumental.mp3
      3.  My Song-vocal.mp3

  Enter number: 1

  Select the VOCAL stem:
      2.  My Song-instrumental.mp3
      3.  My Song-vocal.mp3

  Enter number: 3

  Select the INSTRUMENTAL stem:
      2.  My Song-instrumental.mp3

  Enter number: 2

  Original      : My Song (Remaster).mp3
  Vocal stem    : My Song-vocal.mp3
  Instrumental  : My Song-instrumental.mp3

  Looks good? Press Enter to start, or Ctrl+C to cancel.
```

The clean output is saved as `My Song (Remaster)_clean.wav` in the same folder.

### Command line

```bash
# With stems (best quality)
python declicker.py "song.mp3" --vocal "song-vocal.mp3" --instrumental "song-instrumental.mp3"

# Without stems (spline repair only)
python declicker.py song.mp3

# Dry run — detect and list clicks, no output saved
python declicker.py song.mp3 --dry-run

# Custom output path
python declicker.py song.mp3 --out fixed.wav
```

## Options

| Flag | Default | Description |
|------|---------|-------------|
| `--vocal` | none | Vocal stem for stem-based repair |
| `--instrumental` | none | Instrumental stem for stem-based repair |
| `--threshold` | `5.0` | Amplitude ratio to flag a candidate. Lower = catches more subtle clicks. |
| `--similarity` | `0.70` | Min spectral fingerprint match (0–1). Raise to reduce false positives. |
| `--out` | `<input>_clean.wav` | Output file path |
| `--dry-run` | off | Detect and list clicks only — no file saved |

## Tuning

- If you hear **remaining clicks** → lower `--threshold` (try `4.0`)
- If you hear **musical damage** → raise `--threshold` (try `6.0` or `7.0`) or raise `--similarity` (try `0.80`)
- Output is always **24-bit WAV** for maximum quality
